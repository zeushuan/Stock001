## 第五階段：終極策略選擇

### 三個最終候選

| 策略 | 適用情境 | 全期均值 | 最低 | 跨年度 σ |
|------|---------|---------|------|---------|
| `P0_T1T3` | 最大化獲利、能承受高波動 | +197% | -290% | 12.7 |
| `P0_T1T3+CB30` | 平衡獲利與風控 | +134% | -166% | 12.6 |
| **`P0_T1T3+POS`** ⭐ | **穩定 Edge generator** | **+142%** | **-166%** | **8.9** |

### 推薦選擇邏輯

```
若優先目標是「最大期望值」 → P0_T1T3
若優先目標是「風險控制」 → P0_T1T3+CB30
若優先目標是「穩定 Edge」 → P0_T1T3+POS ⭐ 最佳折衷
```

### 為何 POS 最穩定？

POS 規則本質：**只在策略已被驗證有效的股票上繼續加碼**
- 死亡迴圈股：累積永遠負 → 永不加碼 → 損失與 v7 相同
- 強趨勢股：累積快速為正 → 正常加碼 → 享受所有放大效應
- 震盪股：累積在零附近 → 加碼受限 → 避免死亡迴圈惡化
- **不依賴任何市況假設，純粹基於「策略 自我驗證」**

這是真正的「自適應」設計：策略自己學習哪些股票該加碼。

## 後續問題探討結果

### ✅ 問題 1：POS 動態門檻化（POS5/POS10/POS20）
**結論：原始 POS（門檻 0%）最佳**
```
POS    +141.68%   風報比 0.85 ⭐
POS5   +112.80%   風報比 0.78
POS10  +107.37%   風報比 0.74
POS20  +100.45%   風報比 0.64
```
門檻越嚴 → 獲利下降 + 尾部僅微幅改善 → 風報比下降

### ✅ 問題 2：POS + 停損後重置 (PR)
**結論：均值上升但尾部惡化，整體不如純 POS**
```
POS+PR    +154.79%   最差 -231%   風報比 0.67
POS5+PR   +104.59%   最差 -174%   風報比 0.60
POS10+PR   +99.32%   最差 -177%   風報比 0.56
```
PR 重置打破了 POS 的保護機制 → 失去自我驗證的價值

### ✅ 問題 3：POS + BD（多頭天數）組合
**結論：純 POS 仍最佳**
```
POS         +141.68%   風報比 0.85 ⭐
```
