# 動態 DXY + ML 探討總結

## 1. 動態 DXY 強度門檻 (DXYS{N})

### 設計
不只看 DXY 方向，看力度：
- DXYS0.5：DXY EMA20 比 EMA60 低 0.5% 以上才進場
- DXYS1：低 1% 以上
- DXYS2：低 2% 以上

### 結果

| 模式 | 均值 | 最低 | 風報比 |
|------|------|------|------|
| POS+DXY | +120.51 | -122 | 0.99 |
| POS+DXYS0.5 | +100.77 | -121 | 0.83 |
| POS+DXYS1 | +54.94 | -147 | 0.37 |
| POS+DXYS2 | +11.05 | -98 | 0.11 |

### 結論
**門檻越嚴 → 進場越少 → 獲利下降**

純方向 DXY 過濾已是最佳。原因：
- DXY 大部分時間 EMA20-EMA60 差距 < 1%
- 強度門檻會把「弱美元下行階段」過早排除
- 簡單方向過濾已捕捉所有重要時機

## 2. ML 決策樹預測

### 設計
- 樣本：1243 檔股票
- 特徵：14 個（bh_pct, v7_pct, bull_days, trend_score, rsi, adx 等）
- 標籤：v7→P0 是否改善 > 5%
- 模型：DecisionTree (max_depth=3) + RandomForest (n=100)

### 結果

| 模型 | CV 準確率 |
|------|---------|
| DecisionTree max_depth=3 | **75.6%** ±3.3% |
| RandomForest n=100 | 74.8% ±3.6% |
| 簡單規則 v7>0 | 69.3% |

### 特徵重要性（隨機森林）
```
v7_pct              37.3%  ⭐⭐⭐
bh_pct              15.0%  ⭐⭐
bull_days_pct        8.7%
rsi_avg              8.2%
pyramids_added       7.6%
trend_score          6.7%
其他                 16.5%
```

### 決策樹規則
```
若 v7_pct ≤ 11.67 → 預測「不改善」
若 v7_pct > 11.67 + 波動 < 22 → 預測「改善」
若 v7_pct > 64.15 → 全部「改善」
```

### 高精度負樣本判別 ⭐
```
規則「v7_pct > 0」：
  預測「不改善」的 304 檔中，91.1% 真不改善
  
→ POS 規則本質就是這個（即時版）
→ 證實 POS 設計理念的正確性
```

## 核心洞察

### POS 已抓到 80% 的可預測訊息
- ML 多 6% 準確率來自「事後特徵」（全期 v7_pct, bh_pct）
- 這些特徵在實盤無法事先取得
- POS 用「即時累積收益」近似事後 v7_pct，是最佳即時方案

### ML 為什麼無法超越 POS 多少？
1. 主要預測力來自 v7_pct（事後）
2. 即時可用特徵（bull_days, trend_score）已被 BD 旗標測過，效果有限
3. ML 模型的「準確率提升」≠「策略獲利提升」
4. POS 已是接近資訊上限的即時規則

### 動態 DXY 為何無效？
- DXY 大部分時間在 ±0.5% 範圍內波動
- 強度門檻過於嚴格，誤殺有效弱美元期
- 純方向已是最佳

## 真正剩餘的探索方向

基於前面的發現，**有實質提升潛力**的方向：

### A. 真實財報日整合 ⭐⭐⭐
- 需要 TWSE 觀測站爬蟲
- 公司個別財報日比月份過濾精準
- 預期效果：避財報暴雷，但可能錯過良好財報飆漲

### B. 盤中即時資料 ⭐⭐
- 開盤跳空對策略影響（用收盤價回測無法捕捉）
- 收盤前 30 分鐘決定（時段效應）
- 預期效果：避免跳空虧損，但工程量大

### C. 跨股票 Pairs Trading ⭐⭐
- 同產業兩股做相對強度配對
- 「半導體龍頭買強賣弱」這類策略
- 預期效果：理論上有 alpha，但需要架構性改動

### D. 動態波動率調整 ⭐
- 根據當前波動率動態調整 ATR 倍數
- 已測試過 VA 旗標，效果有限
- 可重新設計（用更精準的波動率指標）
